Backup & Restore

The Settings page (/settings in the web UI) lets you export and import your entire configuration as JSON. Use this to create backups, migrate between instances, or share configurations.

Export

Downloads all monitors, groups, proxies, notification channels, maintenance windows, and status pages as a single JSON file. Two options:

  • Export — includes all data including secrets (passwords, tokens, bearer keys)
  • Export (redacted) — replaces secrets with *** for safe sharing

Via API:

curl -H "X-API-Key: ak_..." https://example.com/api/v1/export > backup.json
curl -H "X-API-Key: ak_..." https://example.com/api/v1/export?redact_secrets=true > backup-safe.json

Import

Upload a previously exported JSON file to restore or transfer configuration between instances. Two modes:

ModeBehavior
MergeCreates new items, skips items that already exist (by name). Existing data is untouched.
ReplaceDeletes all existing monitors, groups, proxies, channels, maintenance windows, and status pages, then imports the file contents.

Via API:

curl -X POST -H "X-API-Key: ak_..." \
  -F "file=@backup.json" -F "mode=merge" \
  https://example.com/api/v1/import

Best Practices

  • Schedule regular exports as part of your backup strategy
  • Use redacted exports when sharing configs between team members
  • Test imports on a separate instance before replacing production data
  • Replace mode is destructive — it deletes everything before importing
  • Import requires admin-level API key access
